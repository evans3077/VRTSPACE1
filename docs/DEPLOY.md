# Deploy guide — Render

This guide walks through getting a VRT SPACE deploy running end-to-end on
Render. Pair it with [`.env.example`](../.env.example) for the full env-var
list and [`STRIPE_SETUP.md`](STRIPE_SETUP.md) for the Stripe section.

---

## 1. First-time setup

### 1a. Repository

1. Fork or clone https://github.com/evans3077/VRTSPACE1
2. Add a Render service via **New → Blueprint** pointing at the repo.
3. The repo's `render.yaml` defines the web service + database. Render will
   prompt you to add the secret env vars listed below.

### 1b. Required env vars (must set before first deploy)

These are the bare minimum. The app will boot without them but most flows
will silently degrade.

| Variable | Why |
|---|---|
| `DJANGO_SECRET_KEY` | Auto-generated by Render via `generateValue: true` |
| `DJANGO_DEBUG=0` | Production |
| `DJANGO_ALLOWED_HOSTS=<your-host>.onrender.com` | Required for CSRF |
| `DATABASE_URL` | Auto-injected from the `vrt-space-db` Render Postgres |
| `APP_BASE_URL=https://<your-host>.onrender.com` | Used in emails + share links |

### 1c. Recommended env vars (set before launching publicly)

| Variable group | Purpose | Without it |
|---|---|---|
| `STRIPE_*` | Subscriptions + credit top-ups | Users can sign up but can't pay |
| `STRIPE_PRICE_STARTER` / `GROWTH` / `AUTHORITY` | Maps Stripe prices to plans | Upgrade buttons say "Request custom scope" |
| `OPENAI_API_KEY` + `GEMINI_API_KEY` + `PERPLEXITY_API_KEY` | P1 AEO precision + P5 public index | AEO falls back to derived scoring; `/aeo-index/` shows "AI engines not yet enabled" |
| `SERPAPI_API_KEY` | Competitor discovery + AI overview | Onboarding step 3 returns empty competitor suggestions |
| `PAGESPEED_API_KEY` | Audit performance scores | Performance score = 0 |
| `EMAIL_HOST` + `EMAIL_HOST_USER` + `EMAIL_HOST_PASSWORD` | Team invites + credit alerts | Emails print to log but never reach users |
| `GOOGLE_OAUTH_CLIENT_ID` + `GOOGLE_OAUTH_CLIENT_SECRET` | "Sign in with Google" | Email/password signup only |
| `SENTRY_DSN` | Error tracking | Errors only logged to Render console |

See [`.env.example`](../.env.example) for the complete list with defaults and
provider-specific notes.

## 2. Stripe configuration

Follow [`STRIPE_SETUP.md`](STRIPE_SETUP.md) end-to-end. The short version:

1. Create products in Stripe with prices `$59 / $149 / $349` monthly.
2. Copy each Price ID into `STRIPE_PRICE_STARTER` / `GROWTH` / `AUTHORITY`.
3. Create three one-time prices at `$10 / $25 / $50` for top-ups.
4. Configure a webhook endpoint at `https://<host>/billing/stripe/webhook/`
   with at least these events:
   - `checkout.session.completed`
   - `customer.subscription.created`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
   - `invoice.payment_succeeded`
   - `invoice.payment_failed`
5. Copy the webhook signing secret into `STRIPE_WEBHOOK_SECRET`.
6. After the first deploy, run:
   ```bash
   python manage.py shell -c "from apps.leads.billing import sync_workspace_plan_catalog; sync_workspace_plan_catalog()"
   ```
   This mirrors the Stripe Price IDs onto each `WorkspacePlan` row so the
   upgrade buttons in the workspace know which Stripe Checkout session to
   open.

## 3. LLM keys (P1 AEO Precision + P5 Public Index)

The AEO workspace runs 15 LLM calls per audit (3 engines × 5 queries) and
the `/aeo-index/` tool runs 9 per lookup (3 × 3). When a key is missing the
engine is silently skipped and a yellow banner explains why.

| Provider | Get your key | Pricing relevant for VRT SPACE |
|---|---|---|
| OpenAI (ChatGPT) | https://platform.openai.com/api-keys | gpt-4o-mini ≈ $0.0002 / query |
| Google Gemini | https://aistudio.google.com/app/apikey | gemini-1.5-flash — free tier covers ~1500 RPD |
| Perplexity (Sonar) | https://www.perplexity.ai/settings/api | sonar ≈ $0.001 / query |

Set:
```
OPENAI_API_KEY=sk-...
GEMINI_API_KEY=AIza...
PERPLEXITY_API_KEY=pplx-...
```

Render auto-redeploys when env vars change.

**Cost ceilings already built in:**

- AEO workspace: credit-gated via `spend_action_credits("aeo")` so users can't
  run unlimited audits.
- Public `/aeo-index/`: hard daily ceiling of **50 fresh lookups** tracked
  via the Django cache (`aeo:index:daily-count`). Stale entries (>30 days)
  are silently re-checked when budget remains.

## 4. Email backend

The app auto-selects the SMTP backend when `EMAIL_HOST` is set. Otherwise it
falls back to the console backend (prints to log).

### SendGrid (recommended free tier)

1. Sign up at https://sendgrid.com (free tier = 100 emails/day).
2. Settings → API Keys → Create API Key with "Mail Send" scope.
3. Verify a sender identity (Single Sender or whole domain).
4. Set in Render env:
   ```
   EMAIL_HOST=smtp.sendgrid.net
   EMAIL_HOST_USER=apikey
   EMAIL_HOST_PASSWORD=<your SG.xxx key>
   EMAIL_PORT=587
   EMAIL_USE_TLS=1
   DEFAULT_FROM_EMAIL=hello@yourdomain.com   (must match the verified sender)
   ```

### Resend (alternative)

1. Sign up at https://resend.com.
2. API Keys → Create API Key.
3. Verify a domain (recommended) or use the `onboarding@resend.dev` sandbox.
4. Set:
   ```
   EMAIL_HOST=smtp.resend.com
   EMAIL_HOST_USER=resend
   EMAIL_HOST_PASSWORD=<re_xxx key>
   EMAIL_PORT=587
   EMAIL_USE_TLS=1
   DEFAULT_FROM_EMAIL=hello@yourdomain.com
   ```

To verify emails are sending after configuring:

1. Sign up a new user → check inbox for a welcome trail (none currently, but the audit-completion email if enabled).
2. At `/workspace/team/`, send yourself an invite — expected email subject `[VRT SPACE] Invite to join <project>`.
3. Watch Render logs for `Failed to send credit alert email` — that's your sign SMTP creds are wrong.

## 5. Sentry

The Django integration only initialises when `SENTRY_DSN` is in the env, so
local dev stays a no-op.

1. Sign up at https://sentry.io (free tier = 5K events/month).
2. Create a Project → choose Django.
3. Copy the DSN.
4. Set:
   ```
   SENTRY_DSN=https://xxx@o123.ingest.sentry.io/xxx
   SENTRY_TRACES_SAMPLE_RATE=0.1
   ```

Sentry tags events with `environment` = `DJANGO_ENV` (dev / staging / production)
so you can filter by deploy environment in the dashboard.

## 6. Background jobs

VRT SPACE supports two job modes:

- **ThreadPoolExecutor (default)** — runs audit / SERP-intel / post-audit
  work inside the web worker process. Free Render tier compatible.
- **Celery + Redis** — needed at scale or when audits start contending with
  request handling. Enable with:
  ```
  AUDIT_USE_CELERY=1
  REDIS_URL=<your-redis>
  CELERY_BROKER_URL=<your-redis>   # optional, falls back to REDIS_URL
  CELERY_RESULT_BACKEND=<your-redis>
  ```
  Then add a separate Render worker service running `celery -A config worker --loglevel=info`.

## 7. Post-deploy checklist

Run through [`TESTING.md`](TESTING.md) sections 1, 2a, 3, and 8 at minimum
before announcing the deploy:

- [ ] Free audit at `/` produces a result page in <90s.
- [ ] Signup → onboarding wizard end-to-end works.
- [ ] `/aeo-index/` lookup completes (or cleanly says "AI engines not yet enabled").
- [ ] Stripe test-mode subscription grants 60 credits to a Starter user.
- [ ] Sentry shows a test event (force one with `from sentry_sdk import capture_message; capture_message("deploy smoke")` in `manage.py shell`).
- [ ] An invite email from `/workspace/team/` actually arrives.

---

## 8. Promoting to live (Stripe)

When the test-mode flow works end-to-end:

1. Stripe dashboard → flip to **Live mode**.
2. Re-create the products + prices (Stripe doesn't share products between modes).
3. Update env vars to the live `pk_live_` / `sk_live_` keys and live Price IDs.
4. Update the webhook endpoint to use the live signing secret.
5. Re-run `sync_workspace_plan_catalog()` on the deployed service.
6. Test with a real card on a Starter plan to confirm credits land before
   announcing publicly.
