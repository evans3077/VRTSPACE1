# VRT Space Agency — Codex Skills Pack

This document defines the skills Codex should use while building and restructuring the VRT Space Agency SaaS project.

The goal is to keep the product simple, high-converting, and scalable while preserving the current core direction:

* **Audit**
* **SEO**
* **AEO**
* **Workspace**
* **Credits / reruns / retention**

Each skill below is a reusable operating mode Codex should follow when working on the project.

---

## Skill 1 — Product Strategy Guardrail

### Purpose

Keep the project aligned with the intended product direction.

### When to use

Use this skill whenever making changes to:

* homepage messaging
* feature scope
* product structure
* public UI
* pricing or positioning

### Rules

* Keep the public product focused on Audit, SEO, and AEO.
* Remove or hide unrelated modules from the main experience.
* Treat the product as a SaaS platform, not a portfolio site.
* Every change should support conversion, clarity, retention, or scalability.
* Avoid adding features that do not directly help the user understand value or make progress.

### Output standard

Any suggested change should answer:

* What problem does this solve?
* How does it improve conversion or retention?
* Does it keep the product simple?

## Skill 2 — Conversion-First UI/UX Design

### Purpose

Shape the interface so visitors understand the value fast and feel motivated to act.

### When to use

Use this skill for:

* homepage design
* landing sections
* CTA structure
* onboarding screens
* pricing pages
* workspace layout

### Rules

* Prioritize clarity over decoration.
* Use a single primary CTA where possible.
* Make the first 3–5 minutes feel valuable.
* Keep spacing generous and layout clean.
* Use strong visual hierarchy.
* Reduce noise, clutter, and competing actions.
* Prefer calm, premium, modern styling.
* Make mobile behavior excellent.

### Design patterns to favor

* Hero with clear promise
* Problem → insight → action flow
* Simple score cards
* Comparison blocks
* Short benefit-led sections
* Visible progress indicators
* Controlled reveal of deeper value

### Avoid

* cluttered dashboards on the public side
* too many buttons
* dense walls of text
* flashy visuals that do not help conversion
* overdesigned sections that slow comprehension

### Output standard

Every screen should help the user feel:

* “I understand this.”
* “This is useful.”
* “I know what to do next.”

## Skill 14 — Product Story Thinking

### Purpose

Keep the entire project aligned to one clear story.

### When to use

Use this skill whenever making strategic decisions.

### Core story

A user enters a site, runs an audit, sees what is wrong, gets SEO and AEO insights, stores the work in a workspace, fixes the site, reruns the analysis, and sees progress over time.

### Rules

* Every feature must serve the story.
* Every screen must move the user forward.
* The product must feel like a growth system.

### Output standard

If a feature does not strengthen this story, it should be simplified or removed.

## Master Rule for All Skills

Whenever Codex works on this project, it should ask:

1. Does this help the user understand value faster?
2. Does this improve conversion, retention, or scalability?
3. Does this keep the experience simple?
4. Does this fit the Audit → SEO → AEO → Workspace loop?

If the answer is no, do not add complexity.
