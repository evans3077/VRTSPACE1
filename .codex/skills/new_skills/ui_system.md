# VRT SPACE — UI System Skill

## 🧠 PURPOSE

Present complex data simply and drive user action.

---

## 🎯 OBJECTIVES

* Display insights clearly
* Improve usability
* Increase conversions

---

## 🏗️ ARCHITECTURE PRINCIPLES

1. UI MUST be fast (minimal JS)
2. Components MUST be reusable
3. Design MUST prioritize clarity over decoration
4. Every page MUST have a clear action

---

## 🧩 CORE COMPONENTS

### Score Card

Displays:

* metric
* value
* trend

---

### Recommendation Card

Contains:

* issue
* impact
* fix
* CTA (“Fix this”)

---

### Dashboard Layout

Sections:

* summary
* insights
* history

---

## 🎨 DESIGN RULES

* consistent spacing
* readable typography
* limited colors

---

## ⚡ INTERACTION RULES

* use HTMX/AJAX
* avoid full page reloads
* smooth updates

---

## 📱 RESPONSIVENESS

* mobile-first design
* adaptive layouts

---

## 💰 CONVERSION ELEMENTS

Include:

* upgrade prompts
* feature locks
* urgency messaging

---

## 📊 DATA VISUALIZATION

Use:

* progress bars
* charts
* trends

---

## ⚠️ EDGE CASES

* no data → show guidance
* errors → user-friendly messages

---

## 📤 OUTPUT EXPECTATIONS

* fast interface
* clear insights
* high engagement

---

## ❌ DO NOT

* overload UI with data
* use heavy JS frameworks unnecessarily
* hide important actions

---

## ✅ SUCCESS CONDITION

Users:

* understand data quickly
* take action easily
* feel the product is premium
