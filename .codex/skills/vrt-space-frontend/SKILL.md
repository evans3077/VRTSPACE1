# VRT Space Frontend

Build pages that are fast, clear, and conversion-oriented.

## Page Workflow

1. Read `references/03_FRONTEND_UI_UX_GUIDE.md` before laying out a new page or section.
2. Read `references/08_LEAD_ENGINE.md` whenever a page contains forms, audit tools, or conversion paths.
3. Build mobile-first layouts with Django templates first; add HTMX or Alpine only where interaction adds value.
4. Keep the homepage and service pages anchored to clear value propositions, trust signals, results, case studies, and strong calls to action.

## Layout Rules

- Optimize for clarity over decoration.
- Use no more than three dominant colors.
- Keep motion minimal and cheap; animation must not harm performance.
- Include a conversion path on every page.
- Prefer deliberate, editorial-feeling sections over generic startup blocks.

## Required Homepage Sections

- Hero with a direct value proposition
- Trust bar
- Services overview
- Results section
- Case-study highlight
- Primary CTA block
